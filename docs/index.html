pgwen

pgwen is a Playwright-based BDD test automation framework. Projects are written as plain Gherkin .feature files plus reusable .meta files — no code, no compilation step. The DSL covers ~125 patterns spanning navigation, locator binding, element actions, text/data bindings, assertions, flow control, and data-driven iteration.

What pgwen is for #

Building executable specifications and automated browser projects that non-developers can author and maintain. The same .feature file you use to describe behaviour is what runs — there's no separate test layer.

  • End-to-end browser automation against any web app (Chromium, Firefox, WebKit)
  • Data-driven runs from CSV / JSON / Gherkin Examples tables
  • Reusable step definitions in .meta files, declared with @StepDef
  • Reports in HTML, JUnit XML, JSON, CSV — failed steps carry a rule-based failure-class badge
  • An interactive REPL for step-by-step exploration against a live browser
  • Opt-in AI-assisted diagnose for 2am failure triage — Claude reads a focused bundle (locator, DOM excerpt, recent git diffs) and proposes a fix

Quick start #

Install pgwen and Playwright as dev dependencies, then install the browser Playwright uses:

npm install --save-dev @pgwen/core playwright
npx playwright install chromium

The AI-assisted scaffolder gives you a working project in one command (requires ANTHROPIC_API_KEY):

npx pgwen new

Or, to write your own scenario from scratch:

Write a feature file at pgwen/features/todo.feature:

Feature: TodoMVC

  Scenario: Add a new todo
    Given I navigate to "https://todomvc.com/examples/react/"
    When  I enter "First task" in the new-todo input
    And   I press the Enter key
    Then  the todo count should be "1 item left"

Bind the locator in pgwen/features/todo.meta:

Feature: Todo bindings

  @StepDef
  Scenario: the new-todo input can be located by class "new-todo"

Run it:

yarn pgwen pgwen/features/todo.feature
$ yarn pgwen pgwen/features/todo.feature

How it runs #

Each .feature file is parsed by Cucumber's Gherkin parser, then composed against the active locator/binding scope (built from your .meta files) and executed step-by-step against a Playwright browser context. The same engine drives the REPL (--repl), so anything you can run in a feature you can also poke at interactively.
