Frequently Asked Questions

This FAQ mirrors the structure of the reference framework's own FAQ so authors moving from the reference framework to pgwen can find familiar answers in the same shape. Each entry either links to the existing pgwen reference doc, or marks the feature as not yet supported with a pointer to the action-items / roadmap track.

Basics #

What is pgwen?

A Playwright-based BDD framework where projects are written as Gherkin .feature files plus reusable .meta files. No TypeScript required to author scenarios — the DSL covers ~300 step patterns spanning navigation, locator binding, element actions, text / data bindings, assertions, flow control, and data-driven iteration. See Introduction.

What are the key features?

  • Rich Gherkin DSL — ~300 step patterns, 34 StepDef annotations.
  • Playwright auto-wait + an internal page-readiness wait.
  • Parallel + dry-run + REPL modes.
  • HTML + JSON + JUnit + CSV reports.
  • AI-assisted diagnose with four selectable providers (Claude / OpenAI / Azure OpenAI / GitHub Copilot).
  • Suggest-only AI fix mode — proposes patches without ever modifying source directly.
  • Pattern-level failure grouping so 500 same-locator failures cost one AI call, not 500.
  • Optional AI project scaffolder (pgwen new) with a 12-question cap + TODO.md checklist for unknowns.

What are the system requirements?

  • macOS / Linux / Windows.
  • Node.js 18+ (uses native fetch, no SDK dependency for any provider).
  • Playwright-supported browsers (Chromium / Firefox / WebKit).
  • No Java runtime required — pgwen is pure TypeScript / Node.

How does pgwen compare with a hand-rolled Playwright test suite?

pgwen trades some of Playwright's raw flexibility for authoring speed and readability. Non-developers can maintain the .feature files; developers only touch the .meta locators (usually one file per project) and configuration. If your team already has a mature Playwright test suite and only developers author it, pgwen doesn't add much. If your team includes QA / BA / ops people who need to own automation, or you're migrating from an existing BDD framework, pgwen keeps the authoring surface familiar and pins the runtime to Playwright.

How mature is pgwen?

Currently 1.0.0-rc.1. ~3000 unit tests, plus a 19-feature / 125-scenario regression pack that runs as a CI gate on every pull request. Project status.

Can I add pgwen to my JS / TS project?

Yes — yarn add -D @pgwen/core (or npm install --save-dev @pgwen/core). See Installation.

Does pgwen have a GUI?

No. All functionality runs via the CLI launcher and the interactive REPL. Reports are rendered as static HTML you open in any browser.

Can I debug my executable specifications?

Yes — use @Breakpoint on a step to drop into the REPL at that point. See Debugging.

Running features #

What are batch and REPL modes?

Batch mode (-b / --batch) exits immediately after execution — use it in CI. REPL mode keeps the session open after the run so you can inspect scope + evaluate ad-hoc steps. See Running the REPL.

How do I run a single feature file?

yarn pgwen path/to/My.feature -b

How do I run a whole directory of features?

yarn pgwen path/to/features/ -b

How do I run multiple feature paths in sequence?

yarn pgwen a.feature b.feature dir/ -b

How do I run features in parallel?

Add --parallel. Parallel scenario execution is also supported. 

yarn pgwen path/to/features/ --parallel -b

Can I stagger the parallel start (ramp-up interval)?

Yes — set pgwen.rampup.interval.seconds in pgwen.conf to stagger worker starts within each batch by N seconds. With maxWorkers=4 and rampup=2: worker 1 at t=0, worker 2 at t=2, worker 3 at t=4, worker 4 at t=6 — then all four run to completion in parallel. Useful when a downstream rate limit or login flow gets slammed by simultaneous starts. 0 / unset = simultaneous up to maxWorkers.

Can I validate my features without executing them?

Yes — pgwen --dry-run resolves every step against the registry + meta StepDefs, reports any unbound references or undefined steps, and never opens a browser. Common short form: yarn pgwen:dryRun.

Can I skip features / scenarios at execution time?

Tag-based filtering works the same as the reference framework: @Ignore, or --tags ~@Wip to exclude. See Tags & filtering.

Can I run only a specific tag?

yarn pgwen --tags @smoke

Reports #

How do I generate reports?

HTML reports generate by default at pgwen/output/reports/html/index.html. Use -f <format> or --format to add others. See Reports.

Can I generate CSV reports?

Yes — both built-in CSV summary reports and config-driven results files via @Results.

Can I generate JUnit-XML reports?

Yes.

Can I generate JSON reports?

Yes — see JSON report. This is the format pgwen diagnose consumes.

Can I generate multiple report formats in one run?

yarn pgwen -f junit,html,json,csv

Data feeds + step composition #

How can I feed CSV data to my features?

See Data-driven runs — same @DataFile / inline CSV patterns Gherkin authors know.

How can I feed JSON data?

Identical interface, JSON file in place of CSV. See JSON feeds.

How do I compose step definitions?

Same as the reference framework — declare a Scenario in a .meta file and annotate with @StepDef. The scenario title becomes the new step phrase callable from features and other meta scenarios. See Authoring features.

Are scenario outlines supported?

Yes — full Scenario Outline + Examples table support.

Are data tables supported?

Yes — multi-row data tables work the same way as the reference framework.

Can I use doc strings?

Yes. Triple-double-quote ("""") docstrings attach to the preceding step.

Can I pass parameters to StepDefs?

Yes — quoted "..." tokens in the step text bind into the StepDef's local scope as positional or named parameters. See DSL → StepDefs for scoping rules.

Can I compare or match content using templates?

Yes — <ref> should match template "<template>" with named <placeholder> tokens that bind on match. See DSL → Template matching.

Can I check that text matches a datetime / number format?

Yes — supported forms: 

<ref> should match datetime format "yyyy-MM-dd"
<ref> should not match number format "#,##0.00"
<ref> at json path "$.dob" should match datetime format "dd/MM/yyyy"
<ref> at xpath "/order/total" should match number format "0.00"
the current url should match datetime format "yyyyMMddHHmm"
the page title should not match number format "0"
<dropdown> text should match datetime format "MMM d, yyyy"
# guards (if / while / until):
<step> if total matches number format "0.00"
I refresh until status matches datetime format "yyyy-MM-dd HH:mm"

Assertions #

Are soft assertions supported?

Yes — three modes: @Hard (fail-fast), @Soft (accumulate + surface at scenario end), @Sustained (accumulate silently). Configured globally via pgwen.assertion.mode or per-step via the annotation.

Can pgwen help me write good Gherkin?

Yes — set pgwen.behavior.rules = "strict" in pgwen.conf. Enforces given-when-then ordering + canonical phrasing.

Browsers + remote execution #

How do I target a specific browser?

pgwen.target.browser in config, or --browser chromium|firefox|webkit at the CLI.

Can I locate elements inside frames?

Yes — locator chaining with in <parent> supports frames + nested elements.

Can I drive multiple browser sessions from one feature?

Partially supported. pgwen handles multiple tabs within a single browser context via I switch to (the )?child tab and the n tabs should be open assertion. Named multi-session swapping (the reference framework's I switch to my X session) is not yet wired.

Can I run pgwen on Docker?

Yes — pgwen/Dockerfile ships in the repo, plus docker-compose.grid.yml for Playwright + WebDriver-style grid scenarios. Note: the base image currently pulls from mcr.microsoft.com/playwright.

Can I run pgwen on BrowserStack or LambdaTest?

Not yet supported. Tracked in project_action_items.md under the operational backlog. Playwright supports both endpoints natively; the integration is mostly credential plumbing + remote URL templates.

Can I run pgwen in CI (Jenkins / Azure Pipelines / GitHub Actions)?

Yes — the migrate CLI already rewrites Jenkinsfile + azure-pipelines.yml and the dry-run / batch modes are CI-friendly. See CLI.

Can I run a headless browser?

Yes — pgwen.web.browser.headless = true in pgwen.conf, or pass --headed=false.

Can I emulate a mobile device in the browser?

Not yet supported via dedicated config. Playwright has it built in; pgwen wires pgwen.web.browser.size and pgwen.web.useragent but no dedicated devices.iPhone 14 preset. Roadmap item.

How do I upload a file?

Same as the reference framework: declare a file-input locator, then use I set the <input> to "<absolute path>" (or a scope ref / config key resolving to a path).

Configuration #

Can I configure runtime settings?

Yes — full HOCON config tree. See Settings.

Is string interpolation supported?

Yes — ${...} resolves against scope, env vars (${env.NAME}), and pgwen.* implicit values (${pgwen.feature.name} etc.). The :masked suffix on config keys redacts the value in logs / reports.

How do I configure logging?

pgwen uses native Node console with structured prefixes — no logback equivalent. Verbosity is controlled by pgwen.console.log in config; CI usually wants "summary", REPL usually wants "all".

Can I work with database data?

Not built in. the reference framework ships SQL bindings; pgwen does not, by design — most enterprise RPA projects interact with apps via UI / API, not direct DB. Projects needing DB access can call out via I execute unix system process (e.g. psql) or a custom JS binding.

AI features (pgwen-only) #

Which AI providers can I use?

Four. Set pgwen.diagnose.ai.provider in config (or --provider at the CLI):

  • claude (default) — uses ANTHROPIC_API_KEY.
  • openai — uses OPENAI_API_KEY.
  • azure-openai — uses AZURE_OPENAI_API_KEY + deployment / api-version env vars.
  • copilot — uses GITHUB_TOKEN; adapter does the Copilot token-exchange internally.

See Diagnose → Providers for the full env-var list.

What does pgwen diagnose do?

Reads a JSON report from a previous run, classifies each failure with a rule-based heuristic (LOCATOR_NOT_FOUND, ASSERTION_FAILED, etc.), and optionally calls the configured AI provider for a structured diagnosis with proposed fixes. See Diagnose.

How do I scaffold a new project?

yarn pgwen new runs an interactive AI-driven scaffolder. Capped at 12 questions total; anything unanswered gets dropped into TODO.md as a checklist. See pgwen new.

Contributing + support #

Where do I report bugs or ask for features?

File against the pgwen project's issue tracker (internal an internal tracker / GitHub — ask your team lead for the link). Include the failing .feature + .meta snippet, the results.json, and the version output (pgwen --version).

Is pgwen open-source?

pgwen is an internal project tracking the reference framework's public surface. the reference framework itself is Apache 2.0.
