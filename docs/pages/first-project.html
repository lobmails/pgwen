Your first project

This walkthrough builds a small project against TodoMVC: add a todo, assert the count, mark it complete.

Project layout #

todo-project/
├── pgwen.conf
├── package.json
└── pgwen/
    ├── features/
    │   ├── todo.feature
    │   └── todo.meta
    └── conf/
        └── profiles/
            └── todo.conf

pgwen.conf #

project { name = "todo-project" }

pgwen {
  baseDir = "pgwen"
  outDir  = "${pgwen.baseDir}/output"

  target {
    browser = "chromium"
    env     = "test"
  }

  web {
    browser { headless = false }       # easier while you're authoring
    capture.screenshots.enabled = true
  }
}

Profile #

pgwen/conf/profiles/todo.conf picks which feature file the todo profile runs:

pgwen {
  launch.options {
    features = [ "${pgwen.baseDir}/features/todo.feature" ]
  }
  outDir = "${pgwen.baseDir}/output/todo"
}

Meta — locator bindings #

pgwen/features/todo.meta:

Feature: TodoMVC bindings

  @StepDef
  Scenario: the new-todo input can be located by class "new-todo"

  @StepDef
  Scenario: the todo count can be located by class "todo-count"

  @StepDef
  Scenario: the first todo item can be located by css ".todo-list li:nth-child(1)"

Feature #

pgwen/features/todo.feature:

Feature: TodoMVC

  Scenario: Add a todo and assert the count
    Given I navigate to "https://todomvc.com/examples/react/"
    When  I enter "First task" in the new-todo input
    And   I press the Enter key
    Then  the todo count should contain "1"

  Scenario: Complete a todo
    Given I navigate to "https://todomvc.com/examples/react/"
    And   I enter "Second task" in the new-todo input
    And   I press the Enter key
    When  I click the first todo item's toggle
    Then  the first todo item should have class "completed"

Run it #

yarn pgwen -p todo
# or directly
yarn pgwen pgwen/features/todo.feature

Dry run #

Validate that every step resolves to a known DSL pattern without launching a browser:

yarn pgwen -p todo -bn

REPL #

Open the interactive REPL against a live browser:

yarn pgwen --repl -p todo

Type Gherkin steps; the engine runs each against the same page. See Running the REPL.
