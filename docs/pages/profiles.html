Configuration profiles

A profile is a config file that overlays pgwen.conf for a specific run target — typically one project process. Profiles live at pgwen/conf/profiles/<name>.conf and are selected with -p <name>.

Layout #

pgwen/
└── conf/
    └── profiles/
        ├── base.conf            # optional — auto-loaded before any named profile
        ├── todo.conf
        ├── signup.conf
        └── reports-only.conf

Example #

A profile typically overrides launch.options to pin which feature(s) it runs, plus the per-process outDir:

# pgwen/conf/profiles/todo.conf
pgwen {
  launch.options {
    features = [ "${pgwen.baseDir}/features/todo.feature" ]
  }
  outDir = "${pgwen.baseDir}/output/todo"
}

Run it:

yarn pgwen -p todo

base.conf inheritance #

If pgwen/conf/profiles/base.conf exists and the active profile is not base, it is loaded first. Use this for common defaults shared by every profile (e.g. report config, results CSV layout).

Environment overlay #

Independent of profiles, pgwen.target.env auto-loads pgwen/conf/env/<env>.conf or .json on top of pgwen.conf. Used for environment-scoped values like base URLs and credentials.

# pgwen/conf/env/test.json
{
  "app": {
    "url": "https://test.example.com",
    "username": "test-user"
  }
}

Extra -c overlays #

-c <file> overlays an arbitrary config file. Commonly used for browser-specific overlays in CI:

yarn pgwen -p todo -c pgwen/conf/browsers/grid.conf

Browser overlay files live in pgwen/conf/browsers/:

# pgwen/conf/browsers/grid.conf
pgwen.web.remote.url = "ws://playwright-grid:9222/playwright"

Final layering order #

  1. pgwen.conf (base)
  2. Env conf for pgwen.target.env
  3. Each -c <file>, in CLI order
  4. base.conf (if active profile ≠ base)
  5. The named profile conf

Each layer's keys override earlier layers'.

Implicit profile bindings #

The active profile name is available as ${pgwen.profile.name}; the active env as ${pgwen.target.env}. Both are commonly used in report keys and run-tagging.
