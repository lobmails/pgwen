DSL reference

pgwen registers ~300 Gherkin step patterns. Steps are matched first-fit against the registry; the first matching pattern handles the step. Locator names can be any free-form noun phrase — the engine resolves them through the active scope.

Navigation #

PatternExample
I navigate to "<url>"I navigate to "https://todomvc.com/examples/react/"
I navigate to <ref>I navigate to base url (resolves from scope)
I refresh the current page / I refresh / I reloadI refresh
I navigate back / I go backI navigate back
I navigate forward / I go forwardI navigate forward
I refresh the current page until <el> is displayedI refresh the current page until the result is displayed
Page-load strategy Navigation steps honour pgwen.web.capabilities.pageLoadStrategy via Playwright's waitUntil. See W3C capabilities.

Locator bindings #

Bind a friendly name to a selector once, then reference it by name in subsequent steps.

Given the new-todo input can be located by class "new-todo"
And   the todo count    can be located by class "todo-count"
And   the items list    can be located by tag name "ul"
Locator typeResolves to
id "submit"[id="submit"]
name "username"[name="username"]
class "error-msg".error-msg
css selector ".my-class > span" / css "..."direct CSS selector
xpath "//div[@class='foo']"XPath selector
tag name "button" / tag "..."tag selector
link text "Home"a:text-is("Home")
partial link text "Hom"a:text("Hom")
javascript "$('.foo:has(.bar)')"jQuery-style → translated to Playwright selectors

Assertions #

PatternDescription
<el> should be displayed / visibleWaits up to pgwen.web.wait.seconds for visibility.
<el> should not be displayedWaits for the element to become hidden.
<el> should be hiddenWait for hidden state.
<el> should be enabled / disabledForm-field interactivity.
<el> should be checked / ticked / uncheckedCheckbox / radio state.
<el> should existWait for element to be attached to the DOM.
<el> should be "<value>"Input value (form fields) or text content.
<el> should contain "<substr>"Substring match on text/value.
<el> should match regex "<pat>"Regex match on text/value.
<n> <el>s should be displayedCount assertion via Playwright locator.count().
<dd> should contain "<option>"Dropdown option presence.

Assertions accept inline annotations like @Timeout('0s') to skip implicit waits or @Message('...') to override the failure message.

Condition vocabulary gaps The following Gherkin condition forms are widely used in production the reference framework projects but are not yet supported by pgwen's evalCondition:
  • <ref> is defined / is not defined
  • <ref> is blank / is not blank
  • <ref> is empty / is not empty
  • <ref> should (not )?be true/false (without quotes)
  • <ref> should (not )?be defined
  • I assert that <step> soft-assert prefix wrapper
These appear in the implementation backlog. Until they land, projects that use them won't pass step resolution on pgwen.

Text bindings #

Capture text from the page or from external sources into a named scope binding.

Given the account name is defined by xpath "//h1" in the page
And   home dir         is defined by system property "user.home"
And   today date       is defined by sql "${db.query}" on "${db.url}"
And   body             is defined by file "${pgwen.baseDir}/fixtures/body.json"
SourcePattern
Literal<name> is "<value>"
Blank / empty / boolean<name> is blank / ... is empty / ... is true / ... is false
XPath<name> is defined by xpath "<p>" in <ref>
JSONPath<name> is defined by json path "$.foo.bar" in <ref>
Regex<name> is defined by regex "(\\d+)" in <ref>
File<name> is defined by file "<path>"
System property<name> is defined by system property "<key>"
SQL<name> is defined by sql "<query>" on "<url>"
JavaScript<name> is defined by javascript "<expr>" · also supports js alias and docstring form
System process<name> is defined by [unix ]system process "<cmd>"

All binding sources accept @Eager or @Lazy — see Annotations.

Capture #

I capture steps read a value at the moment they run and bind the result.

When I capture the text in the result panel as the result
And  I capture the current url as the post-login url
And  I capture the page title as the page title
And  I capture the screenshot as the failure screenshot
And  I capture (.+) attribute of <el> as <name>
And  I capture the value of <el> as <name>
And  I capture the selected option in <dropdown> as <name>
And  I capture the number of <el> elements as the count
And  I capture the PDF text from file "<path>" as <name>

Date & number formatting #

Use the @DateTime or @Number prefix on a format step:

And @DateTime I format Date of Birth from "dd/MM/yyyy" to "yyyy-MM-dd" as date of birth
And @Number   I format the amount    from "#,##0.00"     to "0.00"        as amount cents

# Conditional form
And @DateTime I format X from "yyyy-MM-dd" to "dd/MM/yyyy" as Y if X is not blank

Flow control #

# Inline guards — any step
When I click the cancel button if confirmation is "needed"
And  I refresh the current page if env.PGWEN_RETRY is "true" otherwise I wait

# Sleep
When I wait 2 seconds
And  I wait 500 milliseconds

# Wait for state
When I wait until the result panel is displayed
And  I wait until "${pgwen.outDir}/done.flag" file exists
And  I wait for the result panel
And  I wait for the alert popup
