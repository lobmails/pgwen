Settings reference

pgwen reads configuration from pgwen.conf in the project root, layered with optional profile and environment configs. All settings live under the top-level pgwen HOCON block.

File layout #

A typical project directory:

my-project/
├── pgwen.conf            # base config — this page documents every key
├── package.json          # yarn scripts that invoke pgwen
└── pgwen/
    ├── conf/
    │   ├── browsers/     # chromium.conf, firefox.conf, grid.conf, ...
    │   ├── env/          # test.json, prod.json — selected by pgwen.target.env
    │   └── profiles/     # todo.conf, calculator.conf — selected by -p
    ├── features/         # .feature + .meta files
    └── meta/             # shared meta (e.g. Results.meta)

Config layering #

Files are merged in this order — later layers override earlier ones:

  1. pgwen.conf in the project root (base)
  2. pgwen/conf/env/<pgwen.target.env>.conf or .json (auto-loaded if pgwen.target.env is set)
  3. Each -c <file> CLI argument, in order given
  4. pgwen/conf/profiles/base.conf (if exists and profile ≠ base)
  5. pgwen/conf/profiles/<-p value>.conf (if -p given)

Core settings #

pgwen.target.browser

Playwright browser engine. One of chromium, firefox, webkit. Default: chromium.

pgwen {
  target {
    browser = "chromium"
  }
}

pgwen.target.env

Target environment name. When set, pgwen auto-loads pgwen/conf/env/<env>.conf or .json.

pgwen.target.env = "test"

pgwen.web.browser.headless

Run the browser without a visible window. Default: true. CLI override: --headed.

pgwen.web.browser.size

Viewport size as WIDTHxHEIGHT. Default: 1280x720.

pgwen.web.browser.size = "1920x1080"

pgwen.web.maximize

If true, the viewport is unset and the browser uses its native window size. Default: false.

pgwen.web.capture.screenshots

Capture a screenshot on every step failure and attach it to the HTML report.

pgwen {
  web.capture.screenshots {
    enabled      = true
    duplicates   = false
    highlighting = false
  }
}

pgwen.web.wait.seconds

Default Playwright timeout in seconds for locator-aware steps (waitFor, assertions, etc.). Default: 20.

pgwen.web.assertions

pgwen.web.assertions {
  delayMillisecs = 200      # poll interval inside FluentWait-style assertions
  maxStrikes     = auto     # number | infinity | auto (auto = unlimited within timeout)
}

pgwen.web.throttle.msecs

Slow each browser action down by this many ms — useful when watching the browser run in headed mode. Default: 100.

pgwen.web.useragent

User-Agent string applied to every page in the context.

pgwen.web.suppress.images

Block image requests at the network layer — speeds up runs against image-heavy apps. Default: false.

Launch options #

Most CLI flags can be set persistently in pgwen.launch.options.*. Profile-level conf files (under pgwen/conf/profiles/) commonly override these per-project-process.

KeyTypeDescription
pgwen.launch.options.featureslist<path>Feature files (or directories) to execute. Equivalent to positional CLI args.
pgwen.launch.options.metalist<path>Common .meta files/dirs loaded before any feature.
pgwen.launch.options.inputDatapathCSV or JSON data feed (equivalent to -i).
pgwen.launch.options.tagslist<str>Tag filter expression, e.g. ["@smoke", "~@wip"].
pgwen.launch.options.parallelbooleanRun feature files in parallel.
pgwen.launch.options.dryRunbooleanSkip browser steps; validate step resolution only.
pgwen.launch.options.formatlist<str>Report formats: html, junit, json, results, none.
pgwen.launch.options.reportpathReport output directory. Default: ${pgwen.outDir}/reports.

Behaviour rules #

pgwen.behaviour.rules (or American spelling pgwen.behavior.rules) controls keyword enforcement for @Context/@Action/@Assertion StepDef annotations. strict rejects mismatched keyword/role pairs; lenient allows them. Default: strict.

Failfast #

pgwen.feature.failfast {
  enabled = true        # stop after first failed scenario
  exit    = false       # if also true, exit the whole run after first failed feature
}

State scope #

pgwen.state.level controls how long bindings live. feature (default) keeps them for the whole feature; scenario resets feature-level bindings between scenarios.

Data feed autoTrim #

pgwen.auto.trim.data {
  csv  = true     # default for CSV feeds
  json = false    # default for JSON feeds
}

Masking #

Any setting suffixed :masked is replaced with pgwen.mask.char (default *) in console output and reports.

my-secret:masked = "${env.MY_TOKEN}"

W3C / WebDriver-style capabilities #

Keys under pgwen.web.capabilities.* follow the W3C WebDriver capability vocabulary. pgwen translates the well-known ones to Playwright equivalents at browser-context creation time. Unknown keys are stored on the resolved BrowserConfig.capabilities map for grid-specific use.

W3C / WebDriver-style keyPlaywright equivalentNotes
pgwen.web.capabilities.pageLoadStrategy goto/reload/goBack/goForward(url, {waitUntil}) eager → domcontentloaded, normal → load, none → commit. Read from scope at each step.
pgwen.web.capabilities.acceptInsecureCerts context ignoreHTTPSErrors Boolean. Useful in dev/test against self-signed endpoints.
pgwen.web.capabilities.locale context locale BCP-47, e.g. en-AU.
pgwen.web.capabilities.se:timeZone context timezoneId WebDriver-style grid convention. timezone, e.g. Australia/Sydney.
pgwen.web.capabilities.proxy.* context proxy sslProxy preferred; socks5:// inferred when only socksProxy is set; proxyType=direct disables.
pgwen.web.capabilities.browserName pgwen.target.browser chrome / MicrosoftEdge → chromium. Overrides pgwen.target.browser when both set.
Heads up Unknown capability keys are passed through as raw strings — pgwen never rejects them, but only the keys above produce a real behaviour change in Playwright.

Remote browsers #

pgwen.web.remote.url connects to a remote Playwright endpoint (WebSocket or CDP) instead of launching locally:

pgwen.web.remote.url = "ws://playwright-grid:9222/playwright"

Reports #

pgwen.report {
  overwrite = false      # if false, append a numeric suffix on collision
  suppress {
    meta = true          # hide @StepDef-only feature pages from the index
  }
  results {              # custom CSV outputs — see Reports guide
    fields { ... }
    files  { ... }
  }
}

The full list of report-related keys is documented in the Reports guide.

Diagnose & telemetry #

Every failed step gets a rule-based failure classification attached automatically — see the coloured badge in the HTML report's error panel and the failureClass field in results.json. Classification runs free (no API key, no network) and the classes feed the pgwen diagnose --rules-only CLI.

Two opt-in keys gate the longer-running telemetry collection that the AI-assisted fix track depends on. Both default to off. 

pgwen.diagnose {
  enabled         = false    # reserved — gates future Claude-backed diagnose paths
  history.enabled = false    # write per-failure JSON sidecars under
                             #   <output>/reports/diagnosis-history/
                             # Used to measure classifier false-positive rate.
}

When history.enabled = true, pgwen writes one JSON sidecar per failed leaf step at the end of each run. Filename format: <feature>__<scenario>__<isoStamp>.json; collisions get -2/-3 suffixes so parallel runs don't overwrite each other. Sidecars are local-only — no network call is ever made.

The environment variable PGWEN_AI_DISABLED=1 is a hard kill switch that short-circuits all diagnose-track writes regardless of config. Use it for incident response.

AI-assisted diagnose (opt-in) #

The Claude-backed branch of pgwen diagnose runs automatically when the gates below pass. Invoke pgwen diagnose with no flag for AI mode; pass --rules-only to force the offline path, or --report to produce a telemetry-analysis Markdown summary instead of per-failure diagnosis. Other flags: --no-cache (bypass the response cache), --dry-run (estimate costs without calling Claude), and --config <path> (custom pgwen.conf location).

The --report mode reads <output>/reports/diagnosis-history/*.json and <output>/reports/diagnosis-cache/<aa>/*.json and emits: counts per classifier class × confidence, per AI category × confidence, token totals, machine-proposal + auto_fix_safe tallies, model usage, and — once reviewers annotate history entries with human_action + outcome — the false-positive rate per class. The target threshold below which @pgwen/fix can ship is < 2 %.

The --annotate mode is the review workflow that populates those fields. Running pgwen diagnose --annotate walks every unannotated history entry in stable filename order and prompts the reviewer: 

$ pgwen diagnose --annotate
Scanning pgwen/output/reports/diagnosis-history/
  3 already annotated · 12 pending review

[1/12] Login__Valid-credentials__2026-06-15T10-00-00-000Z.json
  Feature:    Login  (features/Login.feature)
  Scenario:   Valid credentials
  Step:       When I click the submit button
  Error:      locator.click: Timeout 5000ms exceeded
  Classifier: LOCATOR_NOT_FOUND (high)

  Human action? [l]ocator / [a]pp / [n]o action / [o]ther / [s]kip / [q]uit: l
  Outcome? [t]rue positive / [f]alse positive / [a]cceptable / [s]kip / [q]uit: t
  Saved: human_action=fixed_locator outcome=true_positive

Use --limit N to cap the session to N entries. Existing annotations are preserved across runs — only entries without an outcome field are surfaced for review.

pgwen falls back to --rules-only output WITH a notice whenever a gate fails — so a missing key, a disabled flag, or a config-less working directory never produces an empty report. AI mode reads the API key from one of three sources (highest precedence first):

  1. Programmatic override (used by tests).
  2. Environment variable — name configurable, defaults to ANTHROPIC_API_KEY. Recommended; the key never lands on disk.
  3. pgwen.diagnose.ai.api.key in pgwen.conf — MUST be declared with the :masked suffix so the HOCON loader redacts it in reports and logs. pgwen emits a warning when it finds the key here without the masked suffix.

All other AI knobs sit under pgwen.diagnose.ai.*. Every value here has a sensible default; the config is opt-in for overrides. 

pgwen.diagnose.ai {
  enabled  = false                     # master gate (default false)
  provider = "claude"                  # "claude" | "openai" | "azure-openai" | "copilot"

  api {
    keyEnv      = "ANTHROPIC_API_KEY"  # env var pgwen reads the key from
    "key:masked" = "sk-ant-..."        # fallback when the env var is absent;
                                       # the `:masked` suffix is REQUIRED — without it
                                       # pgwen warns and the value can appear in logs.
  }

  # Per-provider model + extras. Only the block for the selected provider is read.
  model            = "claude-sonnet-4-6"   # cross-provider fallback when *.model isn't set
  claude.model     = "claude-sonnet-4-6"
  openai.model     = "gpt-4o-2024-08-06"
  copilot.model    = "gpt-4o"
  azureOpenai {
    model      = "gpt-4o"             # deployment alias also fine here
    resource   = "example-ai"          # <resource>.openai.azure.com — required unless baseUrl is set
    deployment = "gpt4o-prod"         # Azure deployment name — REQUIRED
    apiVersion = "2024-08-01-preview" # REQUIRED
    # baseUrl  = "https://gateway.example/proxy"  # overrides resource for Azure AI gateways
  }

  cost {
    perRunUsd   = 0                    # 0 = unlimited; e.g. set to 2 for $2/run cap
    perDayUsd   = 0                    # 0 = unlimited; e.g. set to 10 for $10/project/day cap
  }

  rate {
    callsPerWindow = 30                # default 30 calls per 5-minute window
    windowMs       = 300000
  }

  pricing {                            # optional — per-model rate overrides on top of defaults
    "claude-haiku-4-5-20251001" {
      output = 4.50                    # only the fields you change need to be set
    }
  }
}

Provider precedence: --provider CLI flag > PGWEN_AI_PROVIDER env (pgwen new only) > pgwen.diagnose.ai.provider config > default "claude". Each provider reads its own API-key env var: ANTHROPIC_API_KEY / OPENAI_API_KEY / AZURE_OPENAI_API_KEY / GITHUB_TOKEN respectively. Unknown provider names fail loud (no silent fallback to a default).

Privacy: a generic PII scrubber runs over every bundle before it reaches Claude (email, JWT, Luhn-validated credit cards, AU/E.164 phone numbers). Projects add domain-specific regexes via extraPatterns; the scrubber is on by default. The PGWEN_AI_DISABLED=1 env hard-stops every AI-track write, including history sidecars.

Bundle enrichment: when AI mode runs, pgwen looks up the failing step's locator binding in the loaded meta files (heuristic — matches the longest binding name that appears as a whole-word match in the step text) and includes: the binding's file:line, selector strategy + value, and ±5 lines of source context. It also runs git log -p --no-merges -n5 -- <feature.feature> <Binding.meta> from the project directory and includes the capped diff in history.recent_diffs. Both enrichments degrade gracefully — a missing git repo, an unparseable meta file, or no name match all yield empty fields rather than a crash. This is what lets Claude tell "the locator changed" from "the app changed" without the on-call dev guessing.

Pattern-level grouping #

AI mode groups failures by a conservative fingerprint (classification class + locator binding file:line + selector strategy + selector value) before calling Claude. One representative goes to Claude on behalf of the whole group; the DiagnoseOutput is then fanned out to every group instance. Same broken locator firing in 500 scenarios costs one Claude call, not 500.

No config knob — grouping is default-on, parity-baseline-safe, and only affects AI mode. To force one Claude call per failure (e.g. for research / measurement) pass --no-pattern-grouping at the CLI. When the locator binding cannot be resolved for a given failure, the fingerprint falls back to a per-failure unique key so the failure is never silently merged with an unrelated pattern.

@pgwen/fix (suggest-only) #

The peer package @pgwen/fix turns Claude's machine_proposal entries into reviewable patches + a self-contained HTML report. Every knob lives under pgwen.fix.* in the project's pgwen.conf and every default is OFF — parity baseline unchanged until the project opts in. 

pgwen {
  fix {
    enabled = false                       # master gate
    reportsDir = "reports/pgwen-fix"      # destination for suggestions/, index.html, history.jsonl
    confidenceMinimum = "high"            # "high" | "medium" — never "low"
    diffMaxLines = 20                     # reject patches exceeding this many +/- lines
    allowedPathPrefixes = ["pgwen/meta/"] # patch must touch only these prefixes
    repeatWindowDays = 14                 # ledger look-back for the circuit breaker
    repeatMaxAttempts = 2                 # max prior `written` outcomes per fingerprint
    githubEnabled = false                 # opt-in PR comment poster (also requires env vars)
  }
}

Settings detail #

KeyDefaultEffect
pgwen.fix.enabledfalseMaster gate. When false, pgwen-fix is a no-op even if invoked.
pgwen.fix.reportsDir"reports/pgwen-fix"Where suggestions land. Resolved relative to the working directory.
pgwen.fix.confidenceMinimum"high"Lowest Claude confidence accepted. "low" is never accepted regardless.
pgwen.fix.diffMaxLines20Total +/- content lines in the unified diff. Headers do not count.
pgwen.fix.allowedPathPrefixes["pgwen/meta/"]Patch file must start with one of these prefixes — keeps suggestions inside meta files and out of src/, CI config, etc.
pgwen.fix.repeatWindowDays14Look-back window for the repeat-fix circuit breaker.
pgwen.fix.repeatMaxAttempts2Max prior written outcomes per fingerprint inside the window before the candidate is rejected as repeat_fix.
pgwen.fix.githubEnabledfalseOpt-in flag for the GitHub PR commenter. Even when true, the commenter is silent unless the env vars below are also set.

Required environment variables #

The optional GitHub PR commenter activates only when ALL of these are set:

Env varPurpose
GITHUB_TOKENPAT or app token with pull-requests: write on the target repo.
PGWEN_FIX_GH_REPOowner/repo e.g. example-org/example-repo.
PGWEN_FIX_GH_PRPull request number to comment on.
GITHUB_API_URLOptional GitHub Enterprise base URL. Defaults to https://api.github.com.

Without those env vars the commenter is a silent no-op — every run still writes the JSON / patch sidecars and the HTML report locally. See the @pgwen/fix README for the end-to-end usage walkthrough.

Runtime self-heal #

The src/heal/ pipeline (Phase 3.6a — pipeline complete, engine hook 3.6b deferred) wires gate → bundle → prompt → AI call → confidence floor → validator → cache → telemetry. Strict opt-in; every default is off. 

pgwen {
  heal {
    enabled = false                      # master gate — only literal "true" turns on
    mode    = "session"                  # "session" | "persist"

    budget {
      maxAttemptsPerStep      = 1
      maxAttemptsPerScenario  = 3
      maxAttemptsPerRun       = 10
      maxUsdPerRun            = 0.50     # 0 = unlimited (matches diagnose BudgetCaps)
    }

    confidence {
      minimum = "high"                   # post-AI floor; lower values rejected
    }

    validation {
      requireExactOneMatch = true        # proposal must locate exactly 1 element
      requireTagMatch      = true        # element tag must match expected_tag
    }

    cooldown {
      seconds = 300                      # refuse to re-heal the same binding too soon
    }

    recentDiffs {
      enabled = false                    # opt-in: include recent git diffs in the prompt
    }

    scrubber {
      extraPatterns = []                 # extra PII regexes (default rules always run)
    }

    report {
      includeInHtml = true               # render heal events as badges
    }
  }
}

Heal is build-complete but production-gated: HealPipeline.attempt() works end-to-end via the MockAdapter, but the engine-side hook into PlaywrightRunner action sites (Phase 3.6b) lands only after @pgwen/fix has been validated in production with a measured FP rate below 2% (strategy §13.4).
