Running the REPL

The REPL gives you a live Gherkin prompt attached to a real browser context. Steps you type execute against the same page; bindings persist between steps; scope is inspectable at any time.

Launch #

yarn pgwen --repl
yarn pgwen --repl -p todo                 # pre-load meta + profile config
yarn pgwen --repl pgwen/features/todo.feature   # pre-load feature meta only

The prompt #

pgwen 1.0.0 [chromium · env=test]
gherkin> Given I navigate to "https://todomvc.com/examples/react/"
[1.2s] passed
gherkin> When I enter "First task" in the new-todo input
[0.4s] passed
gherkin> dump
  base url        = https://todomvc.com/examples/react/
  pgwen.target.env = test
  pgwen.now        = 2026-06-05T01:23:45.678Z
gherkin> exit

Post-execution REPL #

Drop into the REPL automatically at the end of every non-batch run with the executed scenario's scope still in memory. Useful for poking at why a step failed without re-launching: 

$ yarn pgwen pgwen/features/login.feature
…
[1 of 1] FAILED · 5 of 6 steps passed
REPL Console
pgwen> env "username"
  env {
    scope : "feature" {
      username : "test-user"
    }
  }
pgwen> And the username field can be located by id "user"
  [3ms] ✓
pgwen> I click the login button
  [120ms] ✓
pgwen> exit

Step bindings made in the REPL land in a fresh feature frame on top of the inherited scope, so they appear under env "<name>" immediately (filtered env hides the settings layer by default — use env -a "<name>" to include config).

Commands #

CommandEffect
dumpPrint the current scope (all bindings).
scopePretty-print scope as a tree.
helpShow the command list.
historyList previously-executed steps.
clearClear the terminal.
exit / quitLeave the REPL and close the browser.
continueInside a @Breakpoint session: resume execution.

Breakpoints #

Annotate any step with @Breakpoint in a meta file, then run with -d:

yarn pgwen -d -p todo

Execution pauses at the breakpoint and opens an attached REPL with the live scope and page. Type continue to resume.

Interactive mode after run #

Pass -i to open the REPL automatically when a normal run finishes (success or failure):

yarn pgwen -i -p todo

This is handy for poking at the post-run scope without re-executing the feature.

Features #

  • Tab completion on bound locator/scope names
  • Up/down arrow history
  • Inline :masked values are redacted in scope dumps
  • The REPL respects every setting your pgwen.conf declares
Commands vs DSL steps The commands above (dump, scope, help …) are REPL-only — they're typed bare, without a Gherkin keyword. Anything starting with Given, When, Then, And is treated as a real DSL step and goes through the same resolver as a feature run.
