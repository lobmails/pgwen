Tags & filtering

Tag any scenario or feature with @tagname. Tags select scenarios at the CLI via --tags <expr>, or persistently via pgwen.launch.options.tags.

Tag syntax #

@smoke @regression
Scenario: A new customer can subscribe
  ...

@wip
Scenario: Profile editing supports password change
  ...

Feature-level tags apply to every scenario inside the feature.

Filter expressions #

ExpressionSelects
@smokeScenarios tagged @smoke.
~@wipScenarios NOT tagged @wip.
@smoke and @criticalScenarios tagged with both.
@smoke or @regressionScenarios tagged with either.
@smoke and ~@wipSmoke scenarios that aren't WIP.

CLI vs config #

# CLI
yarn pgwen --tags @smoke

# Persistent in pgwen.conf
pgwen.launch.options.tags = [ "@smoke", "~@wip" ]

When both are present, the CLI flag wins.

Reserved tags #

Some tags are reserved annotations (see Annotations): @StepDef, @Context, @Action, @Assertion, @Examples, @ForEach, @DataTable, @Eager, @Lazy, @Breakpoint, @Abstract, @Before*, @After*, @Results. Avoid using these as filter labels.

Common patterns #

  • @smoke — fast, single-happy-path scenarios; run on every CI build.
  • @regression — broader coverage; nightly run.
  • @wip — work-in-progress; always excluded in shared pipelines via ~@wip.
  • @flaky — quarantined; reviewed weekly.

Empty filter #

An empty [] in pgwen.launch.options.tags is treated as no filter — every scenario runs. --tags "" on the CLI is equivalent.
