Debugging

Techniques for figuring out why a step misbehaves.

Watch the browser #

yarn pgwen --headed --slow-mo 200 -p todo

--headed shows the browser window; --slow-mo 200 adds 200 ms between actions so you can follow what's happening. Equivalent settings live under pgwen.web.browser.headless and pgwen.web.throttle.msecs.

Dry-run first #

yarn pgwen -bn -p todo

The two-phase dry run checks step resolution (Phase 1) and then runs the full pipeline without a browser (Phase 2). Undefined steps and binding errors surface immediately.

Drop into the REPL #

yarn pgwen --repl -p todo

Iterate on the failing sequence interactively. See Running the REPL.

Breakpoints #

Annotate the step you want to pause at with @Breakpoint in a meta file, then run with -d:

@StepDef
@Breakpoint
Scenario: I look up order "<ORDER_ID>"
  ...
yarn pgwen -d -p todo

Execution pauses at the breakpoint and opens an attached REPL with the live scope and page. Type continue to resume.

Screenshot on failure #

pgwen.web.capture.screenshots {
  enabled    = true
  duplicates = false
}

Every failed step attaches a screenshot to the HTML report. duplicates = false de-duplicates back-to-back captures of the same DOM state.

Playwright trace #

pgwen.browser.trace = "retain-on-failure"   # off | on | retain-on-failure

Produces a Playwright trace file under ${pgwen.outDir}/traces/. Open it with:

npx playwright show-trace pgwen/output/traces/<run>.zip

Drop a breakpoint into the .meta to debug a specific step #

Pair the trace with a @Breakpoint annotation on the failing step. The runner pauses there in --debug mode and drops you into the REPL with the live page + scope: 

Feature: Account search

  Scenario: open the failing account page
    Given I am on the search page
    @Breakpoint When I search for account "A-EF949C67"
    Then the account detail page should be displayed
$ yarn pgwen --debug pgwen/features/account-search.feature
…
pgwen> highlight the example field
pgwen> env "the account"
pgwen> exit          # resume the run, keep the trace for later

Video #

pgwen.web.capture.video = "retain-on-failure"

Records a video of the run and keeps it on failure. Off by default — videos are heavy.

Console log depth #

pgwen.console.log.depth controls how many StepDef nesting levels appear in console output (default 1). Set to "infinity" while debugging to see every nested step.

Inline locator errors #

pgwen.error.messages.inline.locators = true includes the resolved selector text in error messages, which is useful when a locator binding is suspect.

Dump scope #

Inside the REPL or after a run (-i), type dump to print every binding currently in scope. Mismatch between expected and actual values is usually the root cause.
