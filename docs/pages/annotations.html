Annotations

Annotations are @Name tags that decorate either a scenario (in a .meta file) or an inline step. Scenario-level annotations tell the engine to treat a scenario as a reusable step definition, restrict its keyword, bind it to a data source, etc. Step-level annotations affect a single step's execution semantics.

Scenario-level annotations #

@StepDef

Declares a reusable step. The scenario name becomes the matchable Gherkin pattern; the steps inside become the body.

@StepDef
Scenario: I open the todo app
  Given I navigate to "https://todomvc.com/examples/react/"
  And   the new-todo input should be displayed

@Context / @Action / @Assertion

Restricts which Gherkin keyword can call the StepDef. With pgwen.behaviour.rules = strict (default), mismatches fail loudly:

  • @Context — only callable from Given
  • @Action — only callable from When
  • @Assertion — only callable from Then

@Examples('<path>', where='<predicate>')

Backs a StepDef Outline by an external CSV/JSON file. Optional where= predicate filters rows.

@StepDef
@Examples('${pgwen.baseDir}/input/orders.csv', where="REGION=='NSW' && STATUS=='ACTIVE'")
Scenario Outline: I look up order "<ORDER_ID>"
  Given I navigate to "${app.url}/orders/<ORDER_ID>"
  Then  the result panel should be displayed

@ForEach

Iterates the step body for every record bound by @DataTable or a captured list binding.

@DataTable

Binds an inline Gherkin data table to the scenario's scope, exposed as pgwen.table.record.*.

@Abstract

Marks a StepDef as a placeholder requiring a concrete override before execution. Useful for shared meta libraries that ship abstract operations.

Current state pgwen parses @Abstract but does not yet validate at invocation time that a concrete override exists. The annotation is recognised; enforcement is a tracked backlog item.

@Results / @Results('<name>')

Routes the scenario's outcome to the configured Results CSV reporter. The named variant routes into a specific reporter declared under pgwen.report.results.files.<name>. See Reports — Results CSVs.

@Import('<file>')

Loads a specific meta file into the active scope. Use to control loading explicitly when auto-discovery wouldn't pick the file up.

@Synchronized / @Synchronised

Ensures a StepDef runs serialised across parallel feature executions — useful when the step touches shared resources.

@Parallel

On a Scenario Outline, runs each expanded example in parallel.

@Ignore

Skips the scenario at parse time — it never appears in the run plan.

@ShadowRoot

Used on element locators that need to traverse a Shadow DOM root before applying the selector.

Step-level annotations #

These prefix a single step (in either a feature or a meta StepDef body) and affect that step's execution.

@Eager / @Lazy

Control when a binding step's value is captured.

  • @Eager — the value is resolved at the moment the binding step runs and stored as a literal. Re-running the binding step re-evaluates and replaces the literal.
  • @Lazy — the value is not resolved at bind time. The first reference triggers evaluation; that first-resolved value is then frozen as a literal so every subsequent reference returns the same captured value.

Without either annotation, binding steps that resolve from a runtime source (JS, XPath, JSONPath, regex, SQL, file, system process) re-evaluate on every reference.

# Captured at bind time
Given @Eager current dateTime is defined by js "new Date().toISOString()"

# Captured at first reference, then frozen
Given @Lazy account ref is defined by xpath "//div[@id='ref']" in the page

@Hard / @Soft / @Sustained

  • @Hard — assertion failures fail the step immediately (default behaviour for Then).
  • @Soft — assertion failures are accumulated; the step reports passed and the failure is surfaced at the end of the scenario.
  • @Sustained — like @Soft but even quieter: failures accumulate silently and the run continues without surfacing them inline.

@Try / @Finally

  • @Try — ignore any failure raised by this step and continue.
  • @Finally — always execute this step at the end of the scenario, even if an earlier step failed. Used for teardown (close the browser, clear input, log out).

@Breakpoint

Pauses execution at this step when running with --debug / -d and opens the REPL with the live scope and page. See Breakpoints.

@Message('<text>')

Overrides the default failure message for an assertion step. Supports scope interpolation in the message.

And the receipt total should be "${EXPECTED_TOTAL}"    @Message('Receipt total mismatch: expected ${EXPECTED_TOTAL} but got ${the receipt total}')

@Timeout('<duration>') / @Delay('<duration>')

  • @Timeout — overrides the implicit wait timeout for this step. '0s' means no wait. Format: 5s, 500ms, 2m.
  • @Delay — adds a fixed delay before the step executes.

@DryRun('<value>')

In dry-run mode, substitutes the step's resolved output with the given value. Lets a feature flow execute without hitting external systems.

@Masked

Hides the bound value in console output and report attachments. Replaced with pgwen.mask.char (default *).

@Trim / @IgnoreCase

Modify comparison semantics on assertion steps. @Trim strips surrounding whitespace from both sides; @IgnoreCase performs a case-insensitive compare.

Placement is step-level only (per the official the reference framework annotations docs) — these go inline on the individual assertion step, not on the Scenario / StepDef Gherkin tag header. Example: 

Given the padded value is "   spaced out   "
 Then @Trim the padded value should be "spaced out"

Given the mixed case value is "Hello World"
 Then @IgnoreCase the mixed case value should be "HELLO WORLD"
  And @IgnoreCase the mixed case value should contain "WORLD"

pgwen also accepts the same annotations on a Scenario / StepDef tag header as a back-compat shorthand (the flags propagate to every assertion in the body), but the inline form is the canonical the reference framework-parity placement and is what project authors should reach for first.

@DateTime / @Number

Prefix on date / number formatting steps. The annotation tells the engine which formatter to apply.

And @DateTime I format Date of Birth from "dd/MM/yyyy" to "yyyy-MM-dd" as date of birth
And @Number   I format the amount    from "#,##0.00"     to "0.00"        as amount cents

User tags #

Any @tagname that isn't one of the reserved annotations above is a user tag — used for filtering via --tags. Common examples: @smoke, @regression, @wip, @flaky. See Tags & filtering.

Typo detection at load #

Every PascalCase annotation tag on a Feature: or Scenario: header is validated against the known set at .meta / .feature load time. Typos throw immediately with a location and an edit-distance suggestion: 

$ yarn pgwen pgwen/features/login.feature
…
IllegalStepAnnotationException: Invalid or illegal annotation
[at pgwen/meta/Login.meta:3]: @Stepdef is not a valid pgwen annotation.
Did you mean @StepDef?

Catches the common gotchas: @StepDefs → @StepDef, @Eagers → @Eager, @Stepdef (case mangling) → @StepDef, @Sycnhronized (transposition) → @Synchronized. Lowercase tags (@smoke, @regression) are user filter tags and skip the check.
