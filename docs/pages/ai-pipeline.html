AI pipeline architecture

How pgwen diagnose's AI-backed branch is built — what runs in what order, what gates exist, and how the design keeps the cost down. Read alongside the diagnose CLI guide.

Per-pattern AI flow #

Before the AI pipeline runs, failures are grouped by fingerprint (classification class + locator binding file:line + selector strategy + selector value). One representative per group is sent to Claude on behalf of all members; the resulting DiagnoseOutput is fanned out to every group instance and the per-entry instances list carries downstream to @pgwen/fix. 500 scenarios failing on the same broken locator cost one Claude call, not 500. See Diagnose → Pattern-level grouping for the fingerprint composition + the --no-pattern-grouping escape hatch.

For each group representative, the orchestrator (AiPipeline.runAiDiagnose) walks through every cost lever before talking to Claude. A gate that says "no" short-circuits with a skipped result + a reason — the user-facing report makes it obvious why a call didn't happen.

Cost levers #

Every lever below is on by default; the AI track is opt-in but the cost controls aren't optional.

LeverMechanismTypical effect
1. Rule-based short-circuit High-confidence rule classifications can skip Claude entirely (caller's choice). Eliminates 30–60% of calls.
2. Local response cache SHA-256 of canonical bundle → diagnose output, 7-day TTL. Flaky scenario = 1 call per week, not per failed run.
3. Prompt caching cache_control: ephemeral on system + tool block. ~90% off cached tokens within Anthropic's 5-minute window.
4. Confidence-tier model routing medium → Haiku, low/unknown → Sonnet. Opus opt-in only. Sonnet is ~$0.027/call typical; Haiku is ~$0.009.
5. Output token cap max_tokens = 800 default (DiagnoseOutput ≈ 300–500 tokens). Caps worst-case output spend.
6. Bundle hard cap 25 KB soft / 50 KB hard via binary-search DOM trim. ~6–12k input tokens typical.

Tune everything from pgwen.conf under pgwen.diagnose.ai.* — see Settings → AI-assisted diagnose.

What goes in the bundle #

The focused failure bundle — the JSON payload Claude actually sees — is the §11 core of the strategy. It's tight by construction: a single failing step, a single locator, no unrelated scenarios, no network logs. 

DiagnoseInput
├─ failing         { feature, scenario, step text + keyword, error class + msg }
├─ locator         { binding name, strategy, selector value, file:line, ±5 context }
├─ artifacts       { trace_zip_path, dom_excerpt (≤10 KB), screenshot_path }
├─ context         { target_env, browser, viewport, sibling_scenarios[] }
└─ history         { recent_diffs (git log -p -n5), prior_pgwen_fix_marker }

The locator field gets populated by LocatorIndex:

  1. For each loaded meta file, scan for declarations matching <name> can be located by <strategy> "<value>".
  2. Index by binding name.
  3. For each failing step, pick the longest binding name that appears as a whole-word match in the step text. Avoids "submit" eating "submit button".
  4. Slice ±5 lines around the binding line from the meta file's source.
  5. If no name matches, emit locator: null rather than guess.

The history.recent_diffs field gets populated by GitDiff:

  1. Files passed: the feature file plus the binding file (when LocatorIndex found one).
  2. Command: git log -p --no-merges -n5 -- <files> via child_process.execFile (no shell).
  3. UTF-8-safe byte-cap at 8 KB.
  4. Missing git binary, non-repo directory, exec errors → empty string, never throw.

Both enrichments answer the same real-world question: did the test change or did the app change? The selector + recent commit history tell that story without the on-call dev grepping.

Privacy — PII scrubber #

The scrubber runs before the bundle leaves the local machine and before the cap-trim, so byte budgeting reflects post-scrub content. 

Raw bundle           Scrubber              Post-scrub bundle
─────────────  ──▶  ┌────────────────┐  ─▶  ─────────────
                    │ • JWT          │
                    │ • Email        │
                    │ • Card (Luhn)  │
                    │ • AU mobile    │
                    │ • 1300/1800    │
                    │ • E.164 phone  │
                    │ • extra regex  │
                    └────────────────┘

Default rules are organisation-independent. Projects add domain-specific patterns via pgwen.diagnose.ai.scrubber.extra = ["regex1", "regex2"] — for example an internal customer-number format or an account-ID prefix. Matches are replaced with [REDACTED-<TYPE>]; the marker carries no characters that any other rule would match, so passes are order-safe.

§14 module boundary — @pgwen/fix #

Patching lives in a separate peer package (@pgwen/fix). The structural boundary is enforced by a build-time test that grep-scans src/ for any import of @pgwen/fix — diagnose can never silently invoke fix because the import does not exist.

Status: Both modes are now shipped.

  • Suggest-only mode (default) — reads pgwen diagnose --json-out output and writes structured JSON + unified-diff patches + a self-contained HTML report under reports/pgwen-fix/. No source files or branches are modified.
  • Auto-apply mode (--auto-apply) — per-entry pipeline validateMinimumDiff → detectPriorPgwenFix → applyMachineProposal. Rewrites the line, creates a pgwen-fix/<slug>-<utc> branch, commits. Local only — never pushes, never opens a PR. Activation requires an explicit --auto-apply flag; §18 governance answers gate the production deploy, not the code itself.

The repeat-fix circuit breaker (detectPriorPgwenFix) walks git log --grep=pgwen-fix -- <file> and refuses auto-apply when the same line has been auto-rebound twice — catches the case where the AI's first heal was wrong.

Offline mode — MockAdapter #

Every AI flow (pgwen new, pgwen diagnose, @pgwen/fix) is exercisable with NO Anthropic API key through the bundled MockAdapter.

Opt in by setting --provider mock or PGWEN_AI_PROVIDER=mock. The adapter loads canned responses from a JSON fixture file (inline objects supported for unit tests). Use cases:

  • Local development before an API key is provisioned — run the full flow end-to-end without burning credit.
  • CI — exercise the AI surface in pull requests without sharing keys.
  • Demos — show stakeholders how diagnose / fix / new work without making live calls.
  • Snapshot tests — lock the prompt body shape against drift (see docs/ai-prompts.md for the prompt audit + edit workflow).

Eight golden fixtures ship at fixtures/ai-mock/ — one per Claude DiagnoseCategory, plus a pgwen new interview fixture and a combined router. See fixtures/ai-mock/README.md for the schema.

Mock is not in the production AiProvider enum — it's a deliberate escape hatch gated before resolveProvider(), so production code paths that switch on provider see only real values when mock isn't requested.

Runtime self-heal — Phase 3 #

src/heal/ is a structurally complete locator-rebind pipeline (gate → bundle → prompt → AI call → confidence floor → validator → cache → telemetry) callable end-to-end via MockAdapter. Strict opt-in via pgwen.heal.enabled = true — defaults to off.

Phase 3.6b is deferred: threading HealPipeline.attempt() through PlaywrightRunner action-site timeouts is a separate refactor with material parity risk. The strategy doc (pgwen-ai-heal-strategy.md) gates the merge on @pgwen/fix production validation with measured FP-rate < 2%. Phases 3.7–3.12 (HTML badges, persist mode, @NoHeal annotation, docs, rollout) follow.

Configuration reference #

pgwen.diagnose {
  enabled         = false      # reserved (legacy gate; AI gate is below)
  history.enabled = false      # opt-in: write per-failure sidecars

  ai {
    enabled = false            # opt-in: enable Claude-backed diagnose

    api {
      keyEnv      = "ANTHROPIC_API_KEY"   # env var name (configurable)
      "key:masked" = "sk-ant-..."         # fallback — `:masked` is REQUIRED
    }

    cost {
      perRunUsd = 0            # 0 = unlimited; e.g. 2 → $2/run cap
      perDayUsd = 0            # 0 = unlimited; e.g. 10 → $10/day cap
    }

    rate {
      callsPerWindow = 30
      windowMs       = 300000  # 5 minutes
    }

    pricing {                  # per-model overrides (top of DEFAULT_PRICING)
      "claude-haiku-4-5-20251001" {
        output = 4.50          # negotiated rate, USD per MTok
      }
    }

    scrubber {
      enabled = true           # PII scrubber on by default
      extra   = []             # extra regex patterns per project
    }
  }
}

PGWEN_AI_DISABLED=1 in the environment is a master kill switch — short-circuits every diagnose-track write regardless of config. Use it for incident response.
