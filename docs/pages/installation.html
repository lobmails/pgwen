Installation

This guide walks through installing pgwen from scratch — no prior JavaScript / Node.js setup assumed.

Prerequisites #

Node.js 18 or newer

Check whether you have Node.js already:

node --version

If you see v18.x or newer you're good. If not, download the LTS installer from nodejs.org and run it. npm ships inside Node.js — no separate install needed.

A terminal

  • macOS: Terminal.app or iTerm2
  • Windows: Windows Terminal, PowerShell, or WSL
  • Linux: whatever you already use

(Optional) Yarn or pnpm

Both work interchangeably with npm. Enable via corepack (bundled with Node.js):

corepack enable
yarn --version           # if you want yarn
pnpm --version           # if you want pnpm

The examples below use npm / npx for consistency, but every command has a yarn and pnpm equivalent (shown alongside where relevant).

Add pgwen to your project #

Create a folder for your automation project and install pgwen inside it. Pick one of the three package managers:

mkdir my-automation && cd my-automation

# --- npm ---
npm init -y
npm install --save-dev @pgwen/core playwright

# --- yarn (1.x or berry) ---
yarn init -y
yarn add --dev @pgwen/core playwright

# --- pnpm ---
pnpm init
pnpm add --save-dev @pgwen/core playwright

This installs two packages:

  • @pgwen/core — the pgwen framework
  • playwright — the browser driver pgwen uses (a peer dependency; you install and pin the version)

Install a browser #

Playwright needs its own browser binaries. Install Chromium (recommended for first-time users):

npx playwright install chromium           # npm
yarn dlx playwright install chromium      # yarn
pnpm dlx playwright install chromium      # pnpm

Downloads about 150 MB. Only needed once per machine. To install all three browsers, drop the chromium argument.

On CI agents that run inside the official Playwright Docker image, the browsers are pre-installed and this step is unnecessary.

The pgwen command #

Installation exposes a pgwen binary in node_modules/.bin/. Every package manager has its own way to invoke a local binary:

Package managerInvocation
npmnpx pgwen <args>
yarnyarn pgwen <args>
pnpmpnpm pgwen <args> (or pnpm exec pgwen <args>)

The rest of these docs use npx pgwen … for consistency; substitute yarn pgwen … or pnpm pgwen … freely.

Note on naming. pgwen new (space) is the CLI subcommand — this is what you type as a consumer. There's also an older script name pgwen:new (with colon) that only appears inside the pgwen framework's own package.json — it's shorthand for framework developers running smoke tests against a locally built dist/. If you see pgwen:new in a doc anywhere, mentally translate to npx pgwen new.

Or add scripts to your package.json for convenience:

{
  "scripts": {
    "pgwen":         "pgwen",
    "pgwen:test":    "pgwen -p test",
    "pgwen:dryRun":  "pgwen -bn",
    "pgwen:repl":    "pgwen --repl"
  }
}

Then run npm run pgwen, yarn pgwen, or pnpm pgwen.

Verify #

Confirm everything is wired correctly:

npx pgwen --version

You should see the installed pgwen version (e.g. 1.0.0-rc.1). If not, check that node_modules/.bin/pgwen exists — ls node_modules/.bin/pgwen on macOS/Linux.
